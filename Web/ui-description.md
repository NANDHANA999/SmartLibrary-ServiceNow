# User Interface Design

## Overview

The system uses ServiceNow forms and catalog items as the user interface.

## Features

### Book Form

* Add new books
* Update book status
* View book details

### Request Form

* Select book from reference field
* Submit request
* Track request status

---

## User Interaction Flow

1. User opens request form
2. Selects available book
3. Submits request
4. Receives status update

---

## UI Policies

* Show Issue Date only when approved
* Make required fields mandatory
* Control visibility of fields

---

## Benefits

* Simple interface
* Easy navigation
* Real-time updates
